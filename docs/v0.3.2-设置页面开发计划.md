# v0.3.2 设置页面开发计划

## 一、概述

为 ePNote 添加设置界面，提供复习参数配置和数据目录管理功能。设置数据持久化到本地 JSON 文件，前后端通过 Tauri IPC 读写。

---

## 二、设置页内容（V1）

### 复习设置

| 设置项 | 控件 | 默认值 | 说明 |
|--------|------|--------|------|
| 默认复习题数 | 数字输入 + 加减按钮 | 10 | 进入复习页时的默认题数 |
| 每日推荐上限 | 数字输入 + 加减按钮 | 50 | 每日推荐系统的最大推荐题数 |
| 新题推荐比例 | 滑块 | 0.3 | **(预留)** UI 显示但禁用，V2 接入推荐算法 |

| 显示推荐调试信息 | 开关 | false | **(开发)** 控制前端是否显示推荐的评分详情和理由 |

## 三、配置数据存储架构

### 存储链路

```
SettingsView.vue         用户操作设置项
    │
    ▼
settings Store (Pinia)   内存中的响应式状态（单例，全局共享）
    │  saveSettings()
    ▼
api/settings.ts          调用 Tauri invoke("save_app_settings_comm", {...})
    │
    ▼  ──── IPC 边界 ────
    │
    ▼
settings_c.rs            Tauri 命令处理器
    │  config::save_settings(...)
    ▼
config.rs                Rust serde_json 序列化/反序列化
    │
    ▼
app_config.json          物理文件
```

### 物理存储位置

`dirs::config_dir()/ePNote/app_config.json`

- **Windows**：`C:\Users\<用户名>\AppData\Roaming\ePNote\app_config.json`
- **macOS**：`~/.config/ePNote/app_config.json`
- **Linux**：`~/.config/ePNote/app_config.json`

### 扩展后的数据格式

```json
{
  "root": "D:\\chb\\APPData\\ePNote",
  "settings": {
    "default_review_limit": 10,
    "daily_recommendation_limit": 50,
    "new_question_ratio": 0.3,
    "recommendation_randomness": 1.0,
    "show_debug_info": false
  }
}
```

### 读写策略

- **启动时**：`main.ts` 路由 ready 后调用 `settingsStore.loadSettings()`，通过 IPC 读取配置文件，填充 store
- **修改后**：UI 操作更新 store ref，保存触发 `settingsStore.saveSettings()`，IPC 调用后端覆写文件
- **核心原则**：所有写操作走"先读整文件 → 修改对应字段 → 回写"，避免字段间互相覆盖
- **向后兼容**：旧版配置文件无 `settings` 字段时，`#[serde(default)]` 自动补齐默认值，不报错

## 五、初始化流程改动影响

### 5.1 `src-tauri/src/app/config.rs` — 关键改动

当前 `save_root()` 用 `File::create` 整文件覆盖写入 `{ "root": "..." }`。扩展 `AppConfig` 增加 `settings` 字段后，`save_root()` 必须改为合并写入，初始化向导每次保存数据目录才不会清空已保存的设置。

| 函数 | 改动 |
|------|------|
| `save_root()` | 先读已有配置 → 更新 `root` 字段 → 回写，不覆盖 settings |
| `load_root()` | 不变，仍然只返回 root 路径 |
| `load_settings()` | **新增**，返回 `Option<AppSettings>`（无文件或缺字段时返回默认值） |
| `save_settings()` | **新增**，先读已有配置 → 更新 `settings` 字段 → 回写 |
| `get_config()` | **新增**，返回完整配置（root + settings），供命令层使用 |

### 5.2 `src/main.ts` — 启动流程

当前启动流程：

```
tauri_check_init_default() → setInitialized() → router.push('/review')
```

需要在此流程中增加 settings 加载：

```
tauri_check_init_default() → setInitialized() → settingsStore.loadSettings() → router.push('/review')
```

### 5.3 `src-tauri/src/app/init.rs`

- 第 32 行 `save_root(&ctx.root)`：如果 `save_root` 改为合并写入模式，行为正确
- 第 83 行 `load_root()`：需确保扩展 `AppConfig` 后能兼容旧版只含 `root` 的 JSON（`#[serde(default)]` 解决）
- 第 85 行 `PathBuf::from("data")`：无配置文件时的回退逻辑，不需要修改

### 5.4 `src/mock/data.ts` — 全局状态

`selectedDirectory` 已持有当前数据目录路径，设置页可以直接复用来显示"当前数据目录"。

## 六、后端命令 API 设计

在 `generate_handler![]` 中注册 3 个新命令：

### `get_app_settings_comm`

- **功能**：返回完整配置（root 路径 + 所有设置项）
- **参数**：无
- **返回**：`{ root: String, settings: AppSettings }`

### `save_app_settings_comm`

- **功能**：保存设置到 `app_config.json`
- **参数**：`{ settings: AppSettings }`
- **返回**：`Result<(), String>`

### `open_data_directory_comm`

- **功能**：调用操作系统文件管理器打开数据根目录
- **参数**：无
- **返回**：`Result<(), String>`
- **平台兼容**：Windows `explorer` / macOS `open` / Linux `xdg-open`

## 七、前端类型定义

```typescript
interface AppSettings {
  default_review_limit: number        // 默认复习题数
  daily_recommendation_limit: number  // 每日推荐上限
  new_question_ratio: number          // 新题推荐比例（预留）

interface AppSettingsResponse {
  root: string
  settings: AppSettings
}
```

## 十一、注意事项

- **向后兼容**：`#[serde(default)]` 保证旧版 `app_config.json`（只有 `root`）能正常读取，settings 自动获取默认值
- **`save_root` 覆盖问题**：初始化向导写入新数据目录时，必须读取旧配置保留 settings 字段，避免设置丢失
- **`per_subject_limit = 10`**：推荐算法中最关键的硬编码，V1 将其改为用户可配的"每日推荐上限"配置项，由命令层传入
- **预留项**：新题推荐比例、导出/导入仅展示按钮，用 `disabled` 状态标注"后续开放"
- **`open_data_directory`**：需平台兼容处理 Windows(`explorer`) / macOS(`open`) / Linux(`xdg-open`)
