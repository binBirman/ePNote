# 时间管理设计说明

## 一、设计目标

支撑：

- 复习记录时间戳
- 题目创建 / 删除时间
- **逻辑日**（按"作息时钟"切分，可设置时区与切日时间）
- 复习间隔计算（`due_at` 公式）
- 跨日范围查询（按日历月拉复习序列）

**设计原则**

- DB 存稳定（i64 unix timestamp 秒）
- 业务层用 `Timestamp` 抽象
- 时区 / 切日可由用户设置（`AppSettings.timezone_offset_hours` / `day_cutoff_hour`）
- chrono 仅在 `util/time/` 内部使用

## 二、时间分层模型

| 层 | 类型 | 说明 |
|---|---|---|
| DB | `i64` | Unix Timestamp（秒） |
| Rust 业务 | `Timestamp` | 业务时间类型 |
| Rust 业务 | `LogicalDay` | 逻辑日编号（i32，1-1-1 起） |
| Rust 业务 | `ClockConfig` | 时区 + 切日配置（since v0.5.0） |
| 前端 | `number` / `string` | unix ms / 格式化字符串 |

## 三、核心类型

### `Timestamp`

```rust
pub struct Timestamp(pub i64);
```

唯一业务时间类型，封装 unix 秒。  
**禁止** Repo / DB / 前端直接用 `i64` 表示时间。

### `LogicalDay`

```rust
pub struct LogicalDay(pub i32);
```

逻辑日编号（`num_days_from_ce`），不是日历日、不是时间戳。

定义：按 `ClockConfig` 配置的 offset + cutoff，把 ts 投影到本地"作息日"。

### `ClockConfig`（v0.5.0+ 新增）

```rust
pub struct ClockConfig {
    pub offset_seconds: i64,  // 本地时区相对 UTC 偏移（东正西负）
    pub cutoff_seconds: i64,  // 一天切到第二天的分界秒（0:00=0, 3:00=10800）
}
impl Default for ClockConfig {
    fn default() -> Self {
        Self { offset_seconds: 8 * 3600, cutoff_seconds: 3 * 3600 }
    }
}
impl ClockConfig {
    pub fn from_settings(offset_hours: i32, cutoff_hour: i32) -> Self { ... }
}
```

- **默认** UTC+8 + 03:00 切日（与历史硬编码行为一致）
- `from_settings(offset_hours, cutoff_hour)` 由 `AppSettings` 构造

## 四、逻辑日计算（chrono 内部）

```text
shifted = ts - cutoff
local = shifted + offset
day_index = num_days_from_ce(local.date)
```

SQL 端：

```sql
((r.reviewed_at + ?offset - ?cutoff) / 86400) AS day_bucket
```

Rust `LogicalDay::from_timestamp` 与 SQL 同公式（`util/time/logical_day.rs`）。


## 六、推荐 / 统计的时间语义

| 场景 | 字段 | 说明 |
|---|---|---|
| `StatsView` 月度折线图 | `day_bucket` | 0:00 切日历日（`dayCutoffHour: 0`），与推荐错开 |
| `generate_recommendation` forget_risk | `due_at`, `last_review_at` | 公式用原 `due_at`（按 streak² 计算） |
| `clock_config` 何时用 | 推荐 | 默认 UTC+8 + 03:00；用户设置覆盖 |
| `clock_config` 何时不用 | 月度折线图 | 强制 `cutoff=0` 让 X 轴对齐日历日 |

## 七、DB 时间字段规范

统一 `INTEGER` unix timestamp（秒）。  
字段命名：`created_at` / `reviewed_at` / `updated_at` / `deleted_at` / `last_review_at` / `due_at`。

## 八、chrono 使用规则

- **只允许** 在 `util/time/` 模块内部使用
- 禁止在 `Repo / DB / Service / Command / View` 中直接 `use chrono::*`

## 九、设计总结

- DB 存 `i64` unix 秒
- 业务用 `Timestamp` 抽象
- 逻辑日编号按 `ClockConfig` 计算，**用户可设置时区与切日**
- chrono 仅在 `util/time/`；前端不直接处理时间逻辑
- 图表（前端）按日历日 0:00 切，与"复习/今日"口径（按作息时钟）解耦
