# 错题本系统 · 业务规则说明（V1）

> 本说明是系统业务规则的唯一依据；具体实现应严格遵循。

## 1. 系统定位

个人错题记录与复习工具，以"题目"为核心对象，帮助用户长期、低负担地回顾与巩固曾经做错的题目。

不评价用户能力，不强制学习计划，仅提供合理的复习建议与真实反馈。

## 2. 核心业务对象

### 2.1 题目（Question）
- 一道曾做错、值得复习的题
- 必须至少含一张题目图
- 可关联答案图片与其他资源
- 含静态内容 + 动态复习状态

### 2.2 资源（Asset）
- 资源的视觉载体（图片）
- 从属于题目，不单独管理
- 逻辑删除不立即物理删除资源

### 2.3 复习记录（Review）
- 每次复习生成一条**不可变**记录
- 结果三选一：CORRECT / WRONG / FUZZY
- 历史记录不可修改、不可覆盖

### 2.4 分类信息（Meta）
- 科目、知识点、试卷等
- 修改/删除分类不影响题目存在
- 详见 `database/数据库设计文档.md` §meta 命名空间

## 3. 基本业务行为规则

### 3.1 添加题目
- 仅题目图即可保存
- 其他信息（答案、分类）可后补
- 新题默认 `NEW` 状态（v0.5.0 移除 DUE 后由 NEW 承接"待复习"语义）

### 3.2 浏览与查找
- 支持按科目、状态、复习次数等查找
- 浏览不改变任何复习状态

### 3.3 删除题目
- **逻辑删除**（`question.deleted_at` 设当前时间），可恢复
- 关联资源**不立即物理删除**：文件保留在 `assets/`，前端通过 `RecycleBinView` 展示
- 删除不影响历史复习记录
- 仅"永久删除"时资源才 `move_to_recycle` 物理清理

## 4. 复习行为规则

### 4.1 复习定义
- 一次复习 = 用户对某题的一次回顾与判断
- 每次生成一条独立记录
- 不修改题目静态内容

### 4.2 复习结果
- CORRECT：当前掌握
- WRONG：理解不稳定
- FUZZY：轻度错误

## 5. 复习策略（推荐算法）

### 5.1 推荐原则

按题目状态推荐**最值得复习的一小部分**题；不制定强制学习计划。

### 5.2 推荐优先级（5 步）

`recommend()` 在 `server/review_manager.rs`：所有 5 步都跳过 SUSPENDED 题。

| 步 | 来源 | 含义 |
|---|---|---|
| 1 | `list_by_last_result("WRONG")` | 最近答错的题 |
| 2 | `list_by_state(NEW)` | 全新题（从未复习） |
| 3 | `list_by_last_result("FUZZY")` | 最近模糊的题 |
| 4 | `list_due_questions(now)` | 到期（已过 due_at） |
| 5 | `list_stale_questions(...)` | 长期未复习（兜底） |

每步从候选池取到上限即停。**`due_at` / `stale` SQL 已含 `state != 'SUSPENDED'` 过滤**；步骤 1/3 在业务层另加 `if q.state == SUSPENDED` 短路。

### 5.3 间隔调整（due_at 公式）

`domain::state_machine::QuestionStateMachine::calculate_interval_days` 在每次 `process_review` 内部调用：

```text
stability   = streak_after + 1
difficulty = 1 + wrong_count_after * 0.2

CORRECT:  ceil(stability² / difficulty)
FUZZY:    ceil(stability² / difficulty × 0.5)
WRONG:    1 天
```

示例（wrong_count=0）：
| streak | 1 | 2 | 3 | 4 | 5 |
|---|---|---|---|---|---|
| raw | 4/1 | 9/1 | 16/1 | 25/1 | 36/1 |
| interval (天) | 4 | 9 | 16 | 25 | 36 |

- 错答：streak 清零 → interval = 1 天
- 模糊：streak 减 1（与推荐系统统一），interval 半值
- 答对累计到 `STABLE_THRESHOLD = 3` 时 state 转 STABLE

### 5.4 数量控制

- 每个 subject 独立 limit：`SubjectConfig.recommendation_limit: Option<u32>` 覆盖全局 `per_subject_daily_limit`
- 全局默认 10 条
- 跳过当天推荐不累积惩罚

### 5.5 推荐分池（generate_recommendation）

dev/preview 与每日推荐共用，详见 `architecture/项目结构.md` 与 `server/recommendation.rs`：

1. 池 A：`review_count == 0` 的题，按 `created_at` 升序取 `ceil(limit × new_question_guarantee_ratio)` 条，标 `reason = ["新题保送"]`
2. 池 B：剩余新题 + 全部老题，按 `final_score` 降序取满剩余名额
3. SUSPENDED 全部落选
4. 归档科目全部落选（`exclusion_reason = ["科目已归档"]`）
5. 各 subject 独立执行 1-4

### 5.6 推荐评分公式（`calculate_score`）

```text
final_score = (1 + forget_risk)
            × freshness_bonus
            × last_wrong_bonus
            × error_rate_bonus
            × randomness
```

| 项 | 公式 | 含义 |
|---|---|---|
| `forget_risk` | `log2(passed/expected + 1)`，<br>`expected = (due - last_review) / DAY`，<br>`passed  = (now   - last_review) / DAY` | 遗忘风险：题到 due 之后越久越该推 |
| `freshness_bonus` | `review_count == 0` 时 `max(1, 5 - days×0.25)`；否则 1 | 新题加成，5→1 线性衰减 16 天 |
| `last_wrong_bonus` | `last_result == WRONG` 时 3.0；否则 1.0 | 上次答错加成 |
| `error_rate_bonus` | `1 + error_rate × log2(review_count + 1)` | 错率加成（带样本量平滑） |
| `randomness` | `0.95 + (qid×31 + day/86400) % 1000 / 10000` | [0.95, 1.05) 微抖动，避免同分固定排前 |

### 5.7 推荐理由标签（`generate_reason`）

`reason` 字段是按优先级排序的标签列表。`due_at = None` 不输出"到期/超期"。

| 优先级 | 条件 | 标签 |
|---|---|---|
| 1 | `review_count ≤ 3` | `新题` |
| 2 | `due_at` 存在且 `overdue > 0` | `超期 N 天` |
| 2 | `due_at` 存在且 `overdue == 0` 且 `review_count > 0` | `到期` |
| 3 | `last_result ∈ {wrong, fuzzy}` | `上次出错` |

### 5.8 落选原因标签（`generate_exclusion_reason`）

`EXCLUSION_THRESHOLD = 1.5`。

| 条件 | 标签 |
|---|---|
| `state == SUSPENDED` | `暂停复习` |
| `state == STABLE && overdue == 0` | `已掌握` |
| `forget_risk < 1.5` | `遗忘风险低` |
| `error_rate_bonus < 1.5` | `错误率低` |
| 都不低 | `同科排名低于 N`（N = `subject_rank`） |

没 `score_detail` 时 `forget_risk` / `error_rate_bonus` 默认 0，都 < 1.5 → 同时输出"遗忘风险低"+"错误率低"。

## 6. 复习统计规则

### 6.1 统计目标
统计用于解释系统行为、反馈真实状态，不评价用户表现。

### 6.2 统计维度

| 节 | 项 | 状态 | 实现 |
|---|---|---|---|
| 6.2.1 | 复习次数 / 题目数量 | ✅ | `StatsView` 概览卡 |
| 6.2.1 | 按时间的复习趋势 | ✅ | `StatsView` 月度折线图（按 0:00 切日历日） |
| 6.2.2 | CORRECT/WRONG/FUZZY 比例 | ✅ | `StatsView` |
| 6.2.2 | 模糊单列、不计入准确率 | ✅ | 准确率分子分母均不含 FUZZY |
| 6.2.3 | 高频错误题 | ✅ | `QuestionsView` "只看高频错题"开关（`wrong_count ≥ 3`） |
| 6.2.3 | 未稳定掌握数量 | ✅ | `StatsView` state_counts |
| 6.2.3 | 稳定判定（连续正确 + 时间） | ✅ | `STABLE_THRESHOLD = 3` + `streak²` due_at |
| 6.2.4 | **科目错误率** | ✅ | `StatsView` 分科复习状况表 |
| 6.2.4 | **知识点错误率** | ⏸ **暂缓** | 用户确认不做 |
| 6.2.4 | **长期未改善知识点提示** | ⏸ **暂缓** | 用户确认不做 |
| 6.2.5 | 待复习数 | ✅ | `daily_pending` from `getDailyRecommendation` |
| 6.2.5 | **推荐量变化趋势** | ⏸ 未做 | 月度折线图未画此维度 |

## 7. 用户控制权与系统边界

- 随时跳过复习
- 手动暂停 / 恢复（suspend / recover 流程，详见 `状态转移模型设计文档.md`）
- 不强制完成目标
- 不进行能力评估或排名

## 8. 明确不包含（V1）

- 自动判题
- AI 生成解析
- 多人协作
- 云端同步
- 强制学习计划

## 9. 设计原则

- 题目优先于统计
- 行为记录优先于结果推断
- 可解释性优先于算法复杂度
- 长期可用性优先于短期激励

## 10. 暂缓 / 未做项汇总

| 项 | 原因 | 计划 |
|---|---|---|
| 知识点错误率 | 用户确认 | 暂缓 |
| 长期未改善知识点提示 | 用户确认 | 暂缓 |
| 推荐量变化趋势 | 工时 | 后续 |
| v0.5.0 移除 DUE 后的迁移 | 现有题已就地转 STABLE | 无需额外迁移 |
