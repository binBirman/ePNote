# 错题本系统 · 状态转移模型（v1.0.0+）

每道题在任意时刻处于以下状态之一：

```
NEW        新题（从未复习）
LEARNING   学习中（不稳定）
STABLE     稳定掌握
SUSPENDED  暂停（用户手动）
```

> v0.5.0 起**已移除** `DUE` 状态（之前用"超期"语义，迁移时存量 `DUE` 全部转 `STABLE`）。  
> 没有 FAILED / MASTERED / ARCHIVED——避免过度设计。

## 每个状态业务含义

### NEW（新题）
- 刚录入
- 没有任何复习记录
- 必须至少复习一次

### LEARNING（学习中）
- 最近出现过错误或模糊
- 理解不稳定
- 复习频率较高

### STABLE（稳定掌握）
- 已连续多次答对
- 复习间隔较长
- **不会永久消失**——超期仍会进入推荐

### SUSPENDED（暂停）
- 用户手动干预
- 系统不再推荐，不参与评分
- 可随时恢复

## 状态转移总览

```
            ┌──────────┐
            │   NEW    │  首次复习
            └────┬─────┘
                 ▼
            ┌──────────┐
            │ LEARNING │◄───────────────┐
            └────┬─────┘               │
        连续正确 │  错误/模糊            │
                 │                     │
            ┌────▼─────┐               │
            │ STABLE   │───────────────┘
            └────┬─────┘
        到期未复习（仍可回到 LEARNING 或保持 STABLE，依赖结果）
                 │
        （推荐时由 due_at 触发，不直接进入 LEARNING；用户复习后根据结果切）
```

`SUSPENDED` 是**旁路状态**：

```
任意状态 ── 手动暂停 ──► SUSPENDED
SUSPENDED ── 手动恢复 ──► 原状态（从 meta `system.PreSuspendState` 读；缺省回退 LEARNING）
```

## 详细转移规则

### NEW → LEARNING
**触发**：用户完成首次复习（任意结果）  
**原因**：新题一定不稳定  
**实现**：`state_machine::process_review` 中 `QuestionState::NEW` 分支

### LEARNING → LEARNING  
**触发**：复习结果 = WRONG 或 FUZZY  
**副作用**：
- `correct_streak` 清零（WRONG）或减 1（FUZZY）
- `due_at` 按 streak² 公式重算

### LEARNING → STABLE
**触发**：`correct_streak ≥ STABLE_THRESHOLD (= 3)`，最近复习为 CORRECT  
**原因**：连续答对达到阈值即视为稳定  
**实现**：`state_machine` 中 `QuestionState::LEARNING` + `ReviewResult::CORRECT` 分支

### STABLE → LEARNING
**触发**：任意一次复习结果 = WRONG 或 FUZZY  
**业务含义**："曾经掌握"≠"永久掌握"

### 任意 → STABLE
**触发**：`correct_streak` 从 LEARNING 升到 3（CORRECT 累计）  
**注**：从 STABLE 自身 CORRECT 也加 `correct_streak` 但状态保持 STABLE

### STABLE（持续）
- 状态保持 STABLE，CORRECT 累加 `correct_streak`
- WRONG / FUZZY → 回到 LEARNING

### 任意状态 → SUSPENDED
**触发**：`suspend_question_comm`，用户手动  
**规则**：`due_at = None`，不参与推荐评分

### SUSPENDED → 原状态或 LEARNING
**触发**：`recover_question_comm`  
**步骤**：
1. 读 `meta` 表 `system.PreSuspendState` 拿原状态（suspend 时写入）
2. 缺省回退 `LEARNING`
3. 删 `system.PreSuspendState` meta 行
4. 状态置回原状态，`due_at = now`（立即可被推荐）
5. 保留 `correct_streak` 与 `wrong_count`

## 字段与状态机关系

| 字段 | 用途 |
|---|---|
| `state` | 当前复习状态 |
| `last_review_at` | 判断"距上次复习多久" → forget_risk |
| `last_result` | 最近一次结果（CORRECT / WRONG / FUZZY） |
| `correct_streak` | 连续答对次数；>= 3 升 STABLE；due_at 公式输入 |
| `wrong_count` | 累计错误次数；due_at 公式 difficulty；高频错题筛选 |
| `due_at` | 下次建议复习时间；推荐 `forget_risk` 公式输入 |

> 所有字段都**派生或可重算**。真正不可变的是 `Review` 记录。

## 设计上的"刻意留白"

- **不定义"毕业状态"**：否则用户永远不会再复习
- **不自动进入 SUSPENDED**：控制权在用户
- **不根据统计反向改变状态**：统计是解释，不是驱动
- **不破坏"行为优先"**：state 字段是缓存，可由 Review 记录重算
