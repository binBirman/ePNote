# 推荐算法分池方案

## 算法流程

```
对每个科目（上限 N）：

  Step 1  新题保证数 = ceil(N × new_question_guarantee_ratio)

  Step 2  池A = 该科新题（review_count == 0）
          按 created_at 升序（越早的越优先）
          取前 guarantee 题 → 入选，reason 标记"新题保送"

  Step 3  池B = 该科全部题目 - 池A已选的题（包括池A中落选的新题）
          使用现有评分算法排序，取前 (N - 池A已选数) 题

  Step 4  合并池A已选 + 池B入选 → 该科最终推荐
```

### 2. `src-tauri/src/server/recommendation.rs`

#### `generate_recommendation()` 方法重构

```
原逻辑（第 186-226 行）：   新逻辑：
按科目分组                  按科目分组
↓                          ↓
for 每个科目：              for 每个科目：
  limit = 该科上限            limit = 该科上限
  questions.sort(score ↓)    guarantee = ceil(limit × ratio)
  questions.truncate(limit)  │
  final_questions.extend()   ├ 池A = questions.filter(review_count == 0)
                             │   pool_a.sort(created_at ↑)
                             │   selected_a = pool_a.take(guarantee)
                             │   selected_a 的 reason = ["新题保送"]
                             │
                             ├ 池B = questions - selected_a
                             │   pool_b.sort(score ↓)
                             │   selected_b = pool_b.take(limit - selected_a.len())
                             │
                             └ final_questions.extend(selected_a + selected_b)
```

关键点：
- 新题判定需要 `review_count`，需要在评分循环中记录并传给分组
- 池B 包含池A 落选的新题，它们仍有机会通过评分入选
- 池B 入选的 reason 保持原有的评分理由（"已到复习时间"、"上次回答错误"等）

#### `preview_recommendation()` 方法同步更新

落选原因调整：

| 来源 | 入选标记 | reason | exclusion_reason |
|------|---------|--------|------------------|
| 池A 被选 | selected=true | "新题保送" | 空 |
| 池A 落选但池B 入选 | selected=true | 评分理由（同原有） | 空 |
| 池A 落选且池B 落选 | selected=false | 评分理由 | "新题保送名额已满，评分未达到入选线" |
| 池B（非新题）落选 | selected=false | 评分理由 | "同科排名第X/Y，分数低于入选线" |

涉及改动：
- `PreviewRecommendationItem` 结构体不变（字段已足够）
- 池A 的新题需要在循环中标记 `reason`
- 需要在分组前记录每题 `review_count`，用于池A/池B 分割
- `exclusion_reason` 生成逻辑需要区分池A 落选和池B 落选

## 边界场景

| 场景 | 行为 |
|------|------|
| `ratio = 0` | guarantee = 0，池A 为空，退化为现有纯评分算法 |
| `ratio = 1` | guarantee = N，全部名额给新题（按创建时间排序），池B 不参与 |
| 该科新题 < guarantee | 池A 全选，剩余名额给池B |
| 该科 0 新题 | 池A 为空，全部名额给池B |
| 该科题目总数 < N | selected_a + selected_b ≤ 题目总数，不越界 |
