# 后端结构（Rust / Tauri 2.x）

> v1.0.0+。`项目结构.md` 是顶层总览；本文是后端各模块的详细分层、依赖方向、职责。

## 顶层目录

```
src-tauri/src/
├── main.rs                    # Tauri builder + command 注册
├── lib.rs                     # module 树
├── error.rs                   # AppError
│
├── app/                       # 应用配置 / 启动
├── asset/                     # 图片资源管理
├── command/                   # Tauri IPC 薄壳
├── dao/                       # schema 包装
├── db/                        # SQLite schema + 迁移
├── domain/                    # 纯领域类型
├── log/                       # 日志
├── repo/                      # row → domain 转换
├── server/                    # 业务逻辑
└── util/                      # 通用工具
```

## 分层与职责

依赖方向**自顶向下、无环**：

```
main.rs / command/    →  server/, domain/, app/, db/connection
server/               →  dao/, domain/, repo/, util/, asset/
dao/                  →  db/schema, db/error, repo/, domain/
repo/                 →  domain/
db/schema/            →  无
domain/               →  无（纯类型）
util/                 →  无（叶子）/ chrono 等外部 crate
asset/                →  util/, domain/
app/                  →  无
```

各层职责：

### `app/` — 应用配置
- `appstate.rs`：全局 `AppState`（含 `Mutex<Inner>`，持有 DB 句柄与初始化状态）
- `init.rs`：DataRoot 选择、`save_root` 合并写入、`load_root`、`set_initialized` 流程
- `config.rs`：`AppSettings` 与 `ClockConfig` 结构、`app_config.json` 读写、默认值（`#4464` 时区/切日）
- **不含业务逻辑**

### `command/` — Tauri IPC 薄壳
- 每个文件一个域（`review_c` / `question_c` / `show_list_c` / `settings_c` / `init_c` / `recommendation_c`）
- 仅做参数解析 + 调 server / dao + 序列化返回
- **禁止**写业务逻辑；看到业务代码要挪到 server

### `server/` — 业务逻辑
- `review_manager.rs`：推荐 5 步 + 状态机（process_review / suspend / recover）+ 复习统计
- `question_manager.rs`：题目 CRUD + AssetStore 整合
- `recommendation.rs`：推荐算法（评分公式 + 分池 A/B + reason / exclusion 词条生成）
- `show_question_view.rs`：show_view 查询
- `review_manager_db.rs`：复习日统计（每日推荐状态 / 今日复习记录）

业务层**唯一**允许做：
- 调多个 DAO 组合
- 用 domain 类型 + util/ 工具
- 写决策（推荐优先级、状态机转移）
- 通过 chrono::Datelike 等类型化时间处理

业务层**禁止**：
- 直接 `use rusqlite::*`（用 `db/connection`）
- 写 SQL（写到 `db/schema/`）
- 直接处理 IO 字节（让 `asset/store` 处理）

### `dao/` — schema 薄包装
- 单一职责：把 `db/schema/*.rs` 的纯 SQL 函数包成方法，附带类型转换
- 不做业务判断
- 命名按域：`question_dao` / `review_dao` / `view_dao` / `asset_dao` / `meta_dao` / `recommendation_dao`
- 返回 `Result<T, DbError>` 或 `Result<T, AppError>`

### `db/` — Schema + 迁移
- `connection.rs`：打开 SQLite 连接
- `migrate.rs`：版本化迁移（v1-v11）
- `schema/`：每张表一个文件，含纯 SQL 函数
  - `question_schema.rs`
  - `review_schema.rs`
  - `view_schema.rs`（show_view 视图）
  - `asset_schema.rs`
  - `meta_schema.rs`
- `tests.rs`：迁移单测

### `repo/` — 类型转换
- `primitive.rs`：基础类型（timestamp / logical_day / ids）转换
- `tests/`：row → domain 单测
- **职责**：SQL row → 业务 `domain::*` 类型的转换
- DAO 已经返回 row，repo 在 DAO 内部调用

> 实际项目里 `repo/` 与 `dao/` 高度耦合（DAO 内部调 `repo::*_row_to_domain`）；边界是"SQL 在 schema/、类型映射在 repo/、DAO 是壳"。

### `domain/` — 纯领域类型
- `enums.rs`：`QuestionState`、`ReviewResult`、`MetaKey`、`SystemMetaKey`、`ExtensionMetaKey`
- `question.rs`、`review.rs`、`view.rs`：结构体
- `ids.rs`：`QuestionId` / `ReviewId` / `AssetId`（i64 newtype）

**禁止**：
- 任何 IO（DB / 文件 / 网络）
- 任何 chrono / rusqlite / tauri

### `util/` — 通用工具
- `time/`：`Timestamp`、`LogicalDay`、`ClockConfig`（v0.5.0+ 时区/切日）
- `path/`：Asset 路径规则
- `fs/`：文件系统原子操作
- `log/`：日志

`util/time` 是项目里**唯一**允许 `use chrono::*` 的地方。其它模块要时间用 `Timestamp` 抽象。

### `asset/` — 资源文件管理
- `store.rs`：`AssetStore`（copy / 删除 / 回收）
- `garbage.rs`：回收站扫描
- `path.rs`：路径规则
- `tests/`：单测

### `log/` — 日志
- Tauri 应用日志配置

## 数据流（一个具体例子）

用户点"完成答题" → 前端 `processReview(questionId, 'CORRECT')`：

```
1.  main.ts (前端)
    →  invoke('process_review_comm', { questionId, result })
2.  src-tauri/src/command/review_c.rs:35  process_review_comm
    →  解析 result 字符串到 ReviewResult 枚举
    →  ReviewManager::process_review(qid, result, now)
3.  src-tauri/src/server/review_manager.rs:109  ReviewManager::process_review
    →  校验非 SUSPENDED
    →  QuestionStateMachine::process_review(&question, result, now)
    →  return StateTransition { new_state, streak, wrong, due_at }
    →  QuestionDao::update_state(...) + update_review_fields(...)
    →  ReviewDao::insert(...)
4.  返回 Question → command 层调 question_to_data 转 JSON
5.  前端 ReviewSessionView 拿到 QuestionData，更新 currentQuestion
```

中间**没有**任何 SQL 字符串字面量出现在 server/command/dao 层 — 全部走 schema 纯函数。

## 关键约定

### 1. 类型传递单向

```
db row  →  repo (row_to_domain)  →  domain struct
            →  serde Serialize  →  JSON  →  前端 TS interface
```

不逆向；前端类型 (`src/types/`) 字段名与 Rust domain 序列化名一致（snake_case 风格）。

### 2. 时间统一 `Timestamp`

```
db      →  i64 unix sec
Rust    →  Timestamp
逻辑日  →  LogicalDay (用 ClockConfig 算)
前端   →  拿 unix ms 展示 / 用 ClockConfig-aware 算 day_bucket
```

`util/time/mod.rs` 导出 `Timestamp` / `LogicalDay` / `ClockConfig` / `days_since` / `now_ts` / `range_of_day`。

### 3. 错误传递

```
db error  →  DbError  (sqlx 包装)
dao error →  AppError::Db(...)
server    →  String (给 command 层 + Tauri IPC)
command   →  Result<T, String>
前端     →  try/catch 显示
```

`Result<(), String>` 是 Tauri command 的最简错误表示。

### 4. 资源（图片）路径

- DB 存 `asset.path` 相对路径
- `AssetStore` 在 DataRoot 实际管理 `assets/<qid>.png`
- 物理删除用 `move_to_recycle` 把文件移到 `trash/`
- **唯一**允许直接操作文件 IO 的模块是 `asset/store.rs` 和 `asset/fs.rs`

### 5. 业务流层职责

| 需求 | 在哪写 |
|---|---|
| 用户输入校验 | 前端（form validators） |
| 调 API 包装 | 前端 `api/*.ts` |
| 状态机转移 | `domain/state_machine.rs` 纯函数 + `server/*_manager.rs` |
| 算法评分/排序 | `server/recommendation.rs::calculate_score` |
| 触发副作用（如写缓存） | `server/*_manager.rs` |
| Tauri command 注册 | `main.rs:generate_handler![]` |
| SQL 字符串 | **只**在 `db/schema/` |
