### 依赖方向验证

```
command/  →  server/ + domain/ + db/connection
server/   →  dao/ + domain/ + repo/ + util/
dao/      →  db/schema + db/error + repo/ + domain/
repo/     →  domain/
db/       →  无（叶子）
domain/   →  无（纯类型）
util/     →  无（叶子）/ chrono 等外部 crate
asset/    →  util/ + domain/
```

单向、无环。

## 数据流（用户视角）

```
Init (选择 DataRoot) 
  → 启动主程序 (router push /review)
      ├─ Review (今日推荐列表)
      │    └─ 答题 → ReviewSession
      │                  └─ 完成 → ReviewSummary
      │                                └─ 点题 → QuestionDetail
      │                                                └─ 回收站/统计/设置 入口
      │
      ├─ Question Detail
      │    └─ QuestionsView (题列表, 含高频错筛选)
      │                  └─ 题目详情/编辑
      │
      └─ Settings (调整全局 + 科目池)
```

## 关键设计要点（跨层）

- **三层责任分明**：`db/schema/` 只写 SQL；`dao/` 包装 row；`server/` 业务；`command/` IPC 薄壳
- **依赖方向单向**：高层不反向依赖低层；`domain/` 不依赖 IO
- **类型从 DB 流向前端**：`db row` → `repo (row_to_domain)` → `domain struct` → `serde` 序列化为 JSON → 前端 `api/review.ts` 接住
- **chrono 隔离在 util/time**：repo / db / server / view 都不直接用 chrono
- **Tauri command 是薄壳**：所有 command 调 server，server 调 dao + domain
- **设置持久化**：`app_config.json` 存 root + settings；`AppSettings.subjects` 是 `HashMap<String, SubjectConfig>`，每条 SubjectConfig 含 `archived` + `recommendation_limit: Option<u32>`

详见 `business/业务规则文档.md`、`business/状态转移模型设计文档.md`。
