# 本地数据目录（DataRoot）

> 截至 v0.5.0+。DataRoot 由用户在 `InitView` 选择，存 `app_config.json` 的 `root` 字段。

## 实际目录结构

```
<DataRoot>/
├── db.sqlite                          # 唯一数据库（题目 / 复习 / 资源 / meta / 推荐）
├── app_config.json                    # 系统配置（root + AppSettings；不在 DataRoot 而在 OS 配置目录）
├── .instance.json                     # 初始化标记（首次启动后由 init.rs 写入）
├── assets/                            # 所有图片资源（按 qid 组织）
│   ├── <qid>.png                      # 题目图（QUESTION 类型）
│   ├── <qid>.jpg                      # 任意合法扩展名
│   └── ...
└── trash/                             # 永久删除前的暂存（仅永久删除时使用；回收站逻辑未占该目录）
    └── ...
```

## 字段规范

### 数据库

- `db.sqlite` 单一文件
- 详细 schema 见 `database/数据库设计文档.md`
- 迁移：v1-v11，参见 `database/迁移策略文档.md`

### `assets/` 文件

- 路径在 `asset.path` 字段以**相对 DataRoot** 存储（例 `assets/<qid>.png`）
- 物理文件名 = `<qid>.<ext>`，无目录层级
- 引用形式：`<DataRoot>/<asset.path>`

### 永久删除流程（`trash/`）

`permanently_delete_question` 会把对应 asset 文件从 `assets/` 移到 `trash/`，**当前实现**没有回收机制（`trash/` 留作未来回收站垃圾清理）。**生产路径的"删除题目"是逻辑删除**（设 `question.deleted_at`），不进 `trash/`，仅在 `RecycleBinView` 显示。

## 配置

`app_config.json` 不在 DataRoot 内部，由 `dirs::config_dir()` 决定：

- **Windows**：`%APPDATA%\ePNote\app_config.json`（例 `C:\Users\<User>\AppData\Roaming\ePNote\app_config.json`）
- **macOS / Linux**：`~/.config/ePNote/app_config.json`

```json
{
  "root": "D:\\chb\\APPData\\ePNote",
  "settings": {
    "default_review_limit": 10,
    "per_subject_daily_limit": 10,
    "new_question_ratio": 0.3,
    "new_question_guarantee_ratio": 0.2,
    "recommendation_randomness": 1.0,
    "show_debug_info": false,
    "subjects": {
      "数学": { "archived": false, "recommendation_limit": null }
    },
    "timezone_offset_hours": 8,
    "day_cutoff_hour": 3
  }
}
```

## 路径与迁移

- DB 存 i64 unix timestamp（秒），不存绝对路径
- `asset.path` 存相对路径（`assets/...`），DataRoot 迁移不影响 DB 内容
- 推荐系统不依赖绝对路径
