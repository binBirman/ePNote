# 科目池架构重构方案（v3）

## 概述

将"科目配置"从推荐系统内嵌逻辑提升为用户级配置，存储在 `AppSettings.subjects` 中，实现原子的全量配置保存，并简化推荐引擎的抽象层级。

---

## 1. 数据结构变更

### 1.1 新增类型

```rust
// config.rs
pub struct SubjectConfig {
    pub archived: bool,                    // 默认 false
    pub recommendation_limit: Option<u32>, // None=用全局值, Some(0)=不推荐
}

impl Default for SubjectConfig {
    fn default() -> Self {
        Self { archived: false, recommendation_limit: None }
    }
}
```

### 1.2 AppSettings 变更

| 字段 | 操作 | 说明 |
|------|------|------|
| `default_review_limit` | 保留 | 每次复习会话的默认题数 |
| `daily_recommendation_limit` | **删除** | 替换为 `per_subject_daily_limit` |
| `per_subject_daily_limit: u32` | **新增** | 每科每天推荐默认题数 |
| `subjects: HashMap<String, SubjectConfig>` | **新增** | 科目配置映射 |
| `new_question_ratio` | 保留 | 不变 |
| `recommendation_randomness` | 保留 | 不变 |
| `show_debug_info` | 保留 | 不变 |

### 1.3 向后兼容

- 旧配置无 `subjects` 键 → `serde(default)` → 空 HashMap
- 旧 `daily_recommendation_limit` 被 serde 自动忽略

### 1.4 配置文件示例

```json
{
  "settings": {
    "default_review_limit": 10,
    "per_subject_daily_limit": 10,
    "subjects": {
      "数学": { "archived": false },
      "英语": { "archived": true },
      "408": { "archived": false, "recommendation_limit": 20 }
    }
  }
}
```

## 2. 推荐算法重写

### 2.1 generate_recommendation

```rust
fn generate_recommendation(
    &self,
    subject_configs: &HashMap<String, SubjectConfig>,
    per_subject_default_limit: u32,
    now: Timestamp,
) -> Result<Vec<RecommendedQuestion>, DbError>
```

**逻辑流程：**

```
所有活跃题目 → 评分（calculate_score 不变）
→ 按科目分组（无科目的归入 "未分类"）
→ subject_groups.keys() 即为全部有效科目（无需额外 meta 查询）

for subject, questions in subject_groups:
    cfg = subject_configs.get(subject)
    if cfg.archived → skip
    limit = match cfg.recommendation_limit {
        Some(0) → skip
        Some(n) → n
        None → per_subject_default_limit
    }
    sort questions by score DESC
    take top limit → add to result

// "未分类" 特殊处理（不存在于 subject_configs 中）
if subject_groups 包含 "未分类":
    limit = per_subject_default_limit
    take top limit from "未分类" group

sort result by score DESC
return result
```

### 2.2 get_daily_recommendation 签名

pub fn get_daily_recommendation(
    &self,
    subject_configs: &HashMap<String, SubjectConfig>,
    per_subject_default_limit: u32,

## 3. 命令层变更

### 3.1 recommendation_c.rs

```rust
#[tauri::command]
pub fn get_daily_recommendation_comm(
    state: tauri::State<AppState>,
) -> Result<DailyRecommendation, String> {
    let guard = state.inner.lock().unwrap();
    let conn = match &*guard {
        Some(inner) => &inner.db,
        None => return Err("App not initialized".to_string()),
    };

    let settings = config::load_settings();
    let rs = RecommendationSystem::new(conn);
    rs.get_daily_recommendation(&settings.subjects, settings.per_subject_daily_limit)
        .map_err(|e| e.to_string())
}
```

**关键变化：** 删除 `target_count` 参数，不再调用 `meta_dao.list_all_subjects()`。

### 3.2 settings_c.rs + main.rs

**无需修改**。现有命令已足够，`save_app_settings_comm` 原子保存全部 `AppSettings`（含 `subjects`）。

## 6. 数据流

```
设置页：
  loadSettings()     → getAppSettings()     → {root, settings(含subjects)}
  loadSubjectPool()  → listSubjects()       → DB全部科目
  用户勾选/取消      → toggleSubjectArchive() → subjectConfigs 变更
  保存               → saveSettings()       → 原子写入全部配置

复习页：
  getDailyRecommendation()（无参数）
    → 加载 settings.subjects + per_subject_daily_limit
    → 评分所有题目 → 按科目分组
    → subject_groups.keys() = 全部科目
    → 查询 subject_configs 获取每科策略
    → 过滤 archived + 每科取 Top N
    → 全局排序 → 存入 recommendation 表 → 返回
```

---

## 7. 设计决策

| # | 决策 | 原因 |
|---|------|------|
| 1 | `subjects` 放入 `AppSettings` | 属用户配置，方便导出/重置/同步 |
| 2 | 单一 `save_app_settings_comm` | 原子更新，避免部分失败不一致 |
| 3 | 删除 `SubjectPolicy` | 当前规模不需要额外抽象 |
| 4 | 用 `subject_groups.keys()` 获取科目 | 推荐引擎已分组，避免重复查询 |
| 5 | "未分类" 不存入 `subjectConfigs` | 虚拟节点，与真实科目区分 |
| 6 | `recommendation_limit: Some(0)` = 不推荐 | 无需额外 `enabled` 字段 |
| 7 | `defaultReviewLimit` / `perSubjectDailyLimit` 分离 | 前者控制会话题数，后者控制生成题数 |
